Install · Verify · Troubleshoot

Get the cleanest first run.

Start with a known browser version and System Default output. Then verify the source tab, mastering state, and playback preference one step at a time.

Requirements

  • Chrome or a compatible Chromium-based browser with the required extension APIs.
  • Chrome 116 or newer, as declared in the current manifest.
  • An audible source playing in a normal browser tab.
  • Developer mode only when installing an unpacked build.

Install an unpacked build

  1. Download and extract the source repository or release archive.
  2. Open chrome://extensions.
  3. Enable Developer mode.
  4. Select Load unpacked.
  5. Choose the directory that directly contains manifest.json.

Do not select the parent directory and do not load the GitHub Pages docs/ directory.

First run

  1. Open a tab containing music, video, or another audible source.
  2. Start playback before opening the extension popup.
  3. Select Start Enhance.
  4. Leave system-default playback on System Default for the first test.
  5. Open Studio and use master bypass to confirm an audible difference.

No audio or no audible change

  • Confirm the selected tab is producing sound and is not muted.
  • Stop and start enhancement again after changing the source tab.
  • Use System Default output to eliminate sink-routing issues.
  • Disable other audio enhancement extensions temporarily.
  • Reload ArSonKuPik on chrome://extensions after modifying source files.
  • Test master bypass and a clearly voiced preset at a safe listening level.

Output-device routing

Output enumeration and sink selection depend on browser and operating-system support. If a named device cannot be selected, return to System Default, verify that the device is active in the operating system, and restart the source tab. ArSonKuPik does not need microphone audio for system-default playback.

Performance

Use ECO on low-power or thermally constrained systems. Use TURBO for the full processing configuration. If playback becomes unstable, test with a single source tab, close unnecessary heavy tabs, switch to ECO, and compare CPU behavior with master bypass enabled.

Report a bug

Search existing issues before opening a report. Include the ArSonKuPik version, browser version, operating system, source type, playback preference, exact reproduction steps, expected behavior, actual behavior, and screenshots or console errors when useful.
