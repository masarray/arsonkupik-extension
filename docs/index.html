Privacy is architecture

Your audio is not a data source.

The extension processes the tab you explicitly select. Its runtime has no host permissions, remote code, analytics SDK, account system, or cloud processing path.

Read the privacy policy
  • Local audio path. Tab audio enters Chrome's local media pipeline and Web Audio graph.
  • No recording or upload. Audio content is not retained or transmitted by ArSonKuPik.
  • Local settings only. Presets and preferences are stored in the browser profile.
  • Permission transparency. Every manifest permission is documented in the repository.

Install and verify

Use a release or load the source.

GitHub releases contain a clean extension archive built from the tagged source. Developers can load the repository root directly and run the included validator before packaging.

Install an unpacked build

Best for testing the current source or reviewing a pull request.

  1. Download and extract the repository or a release archive.
  2. Open chrome://extensions.
  3. Enable Developer mode.
  4. Select Load unpacked and choose the project root.
  5. Play audio, open ArSonKuPik, and select Start Enhance.

Validate and package

No production dependency or bundler is required.

git clone https://github.com/masarray/arsonkupik-extension.git cd arsonkupik-extension npm run check npm run package

Common questions

Before you press play.

Does ArSonKuPik upload or record browser audio?

No. The extension is designed to process the selected tab locally through Chrome's tab-capture and Web Audio APIs. The runtime contains no cloud audio transport.

Why does it need tab capture?

Chrome must provide the selected tab's audio stream before an extension can process it. Capture begins only after a user action.

Can it process every tab at once?

The current workflow is intentionally centered on one user-selected source tab so status and control remain clear.

Is this a replacement for DAW mastering?

No. ArSonKuPik is a real-time listening enhancer for browser playback, not an offline mastering and delivery environment.

Where are custom presets saved?

They are stored locally in Chrome extension storage for the current browser profile.
