# Support

## Before opening an issue

- Confirm that Chrome is version 116 or newer.
- Reload the unpacked extension after changing source files.
- Verify that the source tab is actively playing audio.
- Test with **System Default** output routing first.
- Disable other audio-processing extensions to rule out conflicts.
- Include the ArSonKuPik version shown in `manifest.json`.

## Bug reports

Use the bug-report issue template and include exact reproduction steps, expected behavior, actual behavior, browser/OS versions, and screenshots when relevant.

## Feature requests

Explain the listening problem, intended audience, desired workflow, and why existing controls or presets do not solve it.

## Supporting development

Technical support and voluntary project support are separate. Users never need to contribute financially to report a bug, request a feature, use core functionality, or receive public troubleshooting guidance. The optional Indonesia QRIS page is described in [SUPPORT_DEVELOPMENT.md](SUPPORT_DEVELOPMENT.md).

## Security and privacy

Do not use a public issue. Follow [SECURITY.md](SECURITY.md).
