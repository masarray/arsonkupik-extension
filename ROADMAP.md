# Roadmap

This roadmap communicates direction, not a release guarantee.

## Near term

- Harden automated regression checks for state normalization and preset migration.
- Expand release packaging validation for Chrome Web Store submissions.
- Add repeatable performance profiling for ECO and TURBO modes.
- Improve onboarding, permission explanations, and first-run output routing.
- Document objective and subjective listening-test methodology.

## Product direction

- Safer adaptive gain staging across varied streaming loudness.
- More transparent stereo enhancement with stronger mono compatibility guards.
- Better preset portability and versioned migrations.
- Additional accessibility and keyboard workflows in Studio.
- Expanded browser compatibility after equivalent audio-capture APIs are validated.

## Non-goals

- Uploading user audio to a cloud service.
- Adding invasive site access or broad host permissions.
- Replacing professional DAW mastering for production deliverables.
